1. Structured vs. Object-Oriented

Structured/Procedural

Focus:

  • Decomposing problems into Functions (procedures).
  • Focuses on Logic & Steps (Sequence, Selection, Repetition).
  • Data and Functions are often separate.
  • Top-down approach.

The Shift

From what to do, to what things are.

Object-Oriented (OOP)

Focus:

  • Representing problems using Real-World Elements (Objects).
  • Focuses on Data and its Behavior (Methods).
  • Data and Methods are bound together (Encapsulation).
  • Bottom-up approach.

2. The Core Components

Class: The Blueprint

A user-defined data type. It is a logical entity.

class Car {

String color; // Data (State)

void drive() {...} // Methods (Behavior)

}

Object: The Instance

A run-time entity. An instance of a class.

Car myCar;

State: color="Red"

Car yourCar;

State: color="Blue"

3. The Pillars: Abstraction & Encapsulation

Abstraction: Hiding Complexity

Outer Layout/Design. Shows only the essential information to the user and hides the internal implementation details.

Real-World Example (Mobile Phone):

You see the display screen and keypad buttons to dial a number. You only interact with what's necessary.

Achieved using: Interfaces

Encapsulation: Binding Data

Inner Layout/Implementation. Binding data (variables) and the methods that operate on that data into a single unit (Class).

Real-World Example (Mobile Phone):

The internal mechanics of how the keypad buttons and display screen are connected with each other using circuits.

Achieved using: Classes

Data Hiding (Protection)

The mechanism to protect the data from the outside world.

This is often accomplished by making instance variables private or protected (using Modifiers).
