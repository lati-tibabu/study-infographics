Module 1: Packages & Project Structure

1.1 What are Packages?

A package in Java is a way to organize classes logically and avoid name conflicts. Think of packages like folders on your computer that group related files.

Types of Packages

  • Built-in packages (e.g. java.util, java.io, java.lang)
  • User-defined packages (created by developers)

1.2 Creating a Package

The package keyword must be the first line in the Java file. The folder structure must match this name.

// File: src/com/school/student/Student.java
package com.school.student;

public class Student {
    public void displayInfo() {
        System.out.println("Student info displayed.");
    }
}

1.3 Using a Package

Use the import keyword to bring in classes from other packages.

// File: src/com/school/main/Main.java
package com.school.main;

// Importing a user-defined package
import com.school.student.Student;

public class Main {
    public static void main(String[] args) {
        Student s = new Student();
        s.displayInfo();
    }
}

Key Notes:

  • You can import everything in a package: import com.school.student.*;
  • Default package (no package statement) is not recommended.

2.4 Polymorphism

Means "many forms." It allows different classes to be treated as the same type (their parent class), but still perform their own unique behaviors.

class Animal {
    void sound() { System.out.println("Some sound"); }
}

class Dog extends Animal {
    void sound() { System.out.println("Bark"); } // Override
}

class Cat extends Animal {
    void sound() { System.out.println("Meow"); } // Override
}

public class Main {
    public static void main(String[] args) {
        // 'a1' is type Animal, but the object is Dog
        Animal a1 = new Dog();
        
        // 'a2' is type Animal, but the object is Cat
        Animal a2 = new Cat();
        
        a1.sound(); // Bark (Dog's version runs)
        a2.sound(); // Meow (Cat's version runs)
    }
}

Module 3: Common Java Utility Classes

3.1 Scanner Class

Used for getting input from the user (e.g., from the keyboard). Found in java.util.Scanner.

import java.util.Scanner;

public class InputExample {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        System.out.print("Enter name: ");
        String name = sc.nextLine(); // Reads a full line of text

        System.out.print("Enter age: ");
        int age = sc.nextInt(); // Reads an integer

        System.out.println("Hello, " + name + ". You are " + age + " years old.");

        sc.close(); // Always close the scanner
    }
}

3.2 Math Class

Provides static methods for common mathematical operations. No import needed (it's in java.lang).

public class MathExample {
    public static void main(String[] args) {
        double x = Math.sqrt(25);     // 5.0 (Square root)
        double y = Math.pow(2, 3);      // 8.0 (2 to the power of 3)
        double z = Math.random();     // 0.0 to 0.999... (Random number)
        long rounded = Math.round(2.7); // 3 (Rounds to nearest long)
        
        System.out.println("Sqrt: " + x);
        System.out.println("Pow: " + y);
        System.out.println("Random: " + z);
    }
}

3.3 String Class

Strings are immutable objects (they cannot be changed) that represent sequences of characters.

public class StringExample {
    public static void main(String[] args) {
        String s1 = "Java";
        String s2 = "Programming";

        System.out.println(s1.length());         // 4
        System.out.println(s1.concat(" Rocks")); // "Java Rocks" (s1 is still "Java")
        System.out.println(s2.toUpperCase());    // "PROGRAMMING"
        System.out.println(s2.substring(0, 4));  // "Prog" (index 0 up to 4)
        System.out.println(s1.equals("java"));   // false (case-sensitive)
        System.out.println(s1.equalsIgnoreCase("java")); // true
    }
}

Provide object "wrappers" for primitive types (like int, double). This is needed for collections like ArrayList which can only store objects.

Module 4: Practice and Integration

4.1 Mini Project: Student Management

This example combines concepts: user-defined classes (Student), encapsulation (private fields), constructors, and utility classes (Scanner).

package com.school.app;

import java.util.Scanner;

// 1. Encapsulated Class
class Student {
    private String name;
    private int age;

    // 2. Constructor
    Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // 3. Public Method
    void display() {
        System.out.println("Name: " + name + ", Age: " + age);
    }
}

// 4. Main Application Class
public class StudentApp {
    public static void main(String[] args) {
        // 5. Using Scanner
        Scanner sc = new Scanner(System.in);
        
        System.out.print("Enter student name: ");
        String name = sc.nextLine();
        System.out.print("Enter age: ");
        int age = sc.nextInt();

        // 6. Creating an object
        Student s = new Student(name, age);
        s.display();

        sc.close();
    }
}

Module 5: Variables & Access Modifiers

Understanding variables and access control is central to mastering Java. They define how data is stored and who can access it.

5.2 Types of Variables (by Scope)

Variables are classified based on where and how they are declared.

public class Student {
public static String schoolName; // 1. Static private String name; // 2. Instance
public void greet(String greeting) { // 3. Parameter
int marks = 95; // 4. Local System.out.println(greeting + name);
}
}

1. Instance Variables (Non-static)

Belong to an object. Each object gets its own copy (e.g., name, age).

2. Static Variables (Class)

Shared by all objects of the class (e.g., schoolName). Accessed via the class: Student.schoolName.

3. Local Variables

Declared inside a method. Only exist within that method (e.g., marks).

4. Parameters

Variables passed into a method (e.g., greeting).
